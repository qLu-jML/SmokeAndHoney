1

Main Menu

Title screen and game entry point. Boots from MainMenu.tscn with dark warm background and centered UI elements.

Prerequisites:

Launch the game executable. No save file should exist initially.

Test Steps:

  1. Launch game -- Verify dark warm background with subtle glow appears. Logo (80x80px) should be centered in upper portion showing beehive silhouette and "Smoke & Honey" text.
  2. Verify buttons -- Three buttons appear centered below logo: Start (green), Continue (amber), Quit (grey). Buttons properly spaced with no overlap.
  3. Decorative elements -- Amber dividers appear above button group and below. Divider separates Continue from Quit.
  4. Version text -- "v0.1 dev -- Cedar Bend, Iowa" visible at bottom in grey text.
  5. Music playback -- mainTheme.mp3 plays in background, loops seamlessly with warm inviting tone.
  6. Continue button state -- Without save file, Continue button greyed out (disabled), cannot click. Visual feedback clear.
  7. Fade-in animation -- All UI elements fade in smoothly over 0.8 seconds on menu load. Not jarring.
  8. Click Start -- Verify character creator opens with fade transition.
  9. Return to menu -- From character creator, press ESC. Menu returns without state loss.
  10. Click Quit -- Game exits cleanly to desktop, no crash messages.
Expected Results:

Logo loads cleanly. Three buttons readable, properly spaced. Version text visible. Music plays continuously. Continue disabled without save. Fade-in smooth. Start transitions to character creator. Quit exits successfully.

Edge Cases:

If logo PNG fails, fallback text "Smoke & Honey" appears. If music missing, game starts without audio errors. Button hover states highlight clearly.

Character Creator

Name and backstory selection screen on first new game. Centered panel (260x130) with multi-step progression.

Prerequisites:

Click Start button on main menu with no existing character.

Test Steps:

  1. Panel appears -- Character creator panel fades in centered on screen (approximately 260x130 pixels).
  2. Name field active -- Name field accepts text input. Cursor visible blinking in field.
  3. Type valid name -- Type name (e.g., "Sarah Chen"). Maximum 18 characters enforced.
  4. Validate minimum -- With less than 2 characters, press E/Enter. Error message appears for 3 seconds: "Your name must be at least 2 characters!"
  5. Enter valid name -- Type valid name with 2+ characters (e.g., "Jo").
  6. Advance to backstory -- Press Tab or S. Name field no longer accepts input, next section highlights.
  7. Cycle backstories -- Press A and D to cycle three options: Newcomer (no experience), Hobbyist (some home experience), Farmer (multi-season farming).
  8. Description updates -- Description text updates for each backstory showing flavor and gameplay context.
  9. Return to name -- Press W. Name still present. Section focuses on name field again.
  10. Return to backstory then confirm -- Press Tab/S to backstory, Tab/S again to Confirm. Press E/Enter to confirm.
  11. Confirm transitions -- Game transitions to home property with fade effect. Character stored in PlayerData.
Expected Results:

Name persists between sections. Backstory cycles through three options. Confirmation triggers game start with fade. PlayerData stores character_name, backstory_tag, character_created=true. Default pronouns set to they/them. Home property loads without errors.

Edge Cases:

Backspace deletes characters one at a time. Maximum 18 characters enforced silently. Empty/single character triggers validation error. Rapid Tab/S presses don't skip sections. ESC from creator returns to main menu and resets creator state.

Player Movement & Controls

Character movement mechanics, hotbar interaction, mode switching, and context-sensitive actions.

Prerequisites:

Complete character creator and enter gameplay on home property.

Test Steps:

  1. 8-directional movement -- Press WASD individually and combined (W+D for diagonal). Smooth movement in all 8 directions (120 px/sec speed).
  2. Run mode -- Hold Shift while moving. Speed increases to ~210 px/sec (1.75x).
  3. Hotbar cycling -- Scroll mouse wheel up/down. Hotbar cycles through 10 slots. Active slot highlighted with colored border.
  4. Direct slot selection -- Press number keys 1-9. Each key selects corresponding slot. Highlight changes immediately.
  5. Till mode toggle -- Press T. "Till" label appears above player. Press T again, label disappears.
  6. Plant mode toggle -- Press F. "Plant" label appears. Press F again, disappears.
  7. Reset to normal -- With mode active, press 0. Mode resets to Normal, both labels gone.
  8. Interact with chest -- Walk near storage chest, press E. Chest inventory opens in grid overlay.
  9. Rotate structure -- Place hive stand (later section), press R near it. Structure rotates 90 degrees visually.
  10. Sleep advance day -- Press Z. Sleep/day advance triggers. Day counter increments by 1.
Expected Results:

Smooth 8-directional movement with consistent speed. Run speed properly increased with Shift. Hotbar cycling/selection works correctly. Mode labels appear/disappear on toggle with no latency. All mode toggles instant. Interaction priority: harvest yard > merchant > hive > NPC > flower > item. Z key triggers day advance.

Edge Cases:

No movement when pause menu open. Heavy items show floating sprite above player. No movement during minigames/cutscenes. Double mode key press doesn't double-toggle. Diagonal movement same speed as cardinal.

HUD (Heads-Up Display)

On-screen game information display showing energy, hotbar, weather, and status information in real-time.

Prerequisites:

In gameplay on home property or any location.

Test Steps:

  1. Energy bar location -- Honey energy bar visible upper-left corner with golden fill and honeycomb texture overlay.
  2. Energy visual feedback -- Bar shows drip/honeycomb pattern. As energy decreases, bar shrinks right to left proportionally.
  3. Energy color coding -- Monitor bar color during gameplay: Gold >45%, Amber 20-45%, Dark red <20%.
  4. Hotbar display -- Hotbar at bottom center with 10 visible slots. Each shows item icon clearly plus stack count (e.g., "5x").
  5. Active item label -- Name of selected item appears below hotbar (e.g., "Hive Tool", "Empty" for no item).
  6. Weather label -- Weather condition displayed on HUD (e.g., "Sunny", "Rainy"). Updates when weather changes on new day.
  7. Info panel toggle -- Press Tab. Panel appears showing: season, month, day, time, weather, money, honey stores, level, XP.
  8. Close panel -- Press Tab again. Panel closes, HUD returns to normal.
  9. Smoke indicator -- Use smoker on hive, then inspect. "SMOKED" orange label appears on HUD.
  10. Standing tier -- Standing tier displayed on HUD (Stranger/Neighbor/Known/Trusted/Respected). Updates as standing increases.
  11. Season banner -- Use dev mode to reach season boundary. Banner fades in (0.5s), holds (2.5s), fades out (0.8s). Correct seasonal colors.
Expected Results:

All HUD elements update in real-time. Energy drains on actions (inspect=-10, chop=-15), refills after sleep. Tab panel shows accurate values. Season banner displays with smooth fade and correct colors (green/Spring, gold/Summer, orange/Fall, blue/Winter). Smoke indicator appears when smoker active, disappears on inspection close.

Edge Cases:

Dev panel (G key) shows additional hive stats and day/month buttons. Hotbar hidden during specific minigames only. Energy bar clamped 0-100%. Tab panel doesn't persist when opening other overlays.

Year rollover after day 224 resets without errors. Month boundary updates all dependent systems. Time 6:00 AM to ~10:00 PM daily, resets on sleep.

Weather System

8-state weather affecting gameplay, visuals, and foraging. Iowa-based seasonal probabilities.

Prerequisites:

In gameplay. Multiple days of progression for weather variation.

Test Steps:

  1. Check weather display -- Press Tab. Current weather shown (e.g., "Sunny", "Rainy").
  2. Weather progression -- Advance several days (press Z). Weather changes day to day, not identical every day.
  3. Rainy restrictions -- On Rainy day, try to inspect hive. Toast appears: "Too wet to open hives -- wait for dry weather!"
  4. Cold restrictions -- On Cold day, try to inspect. Toast: "Too cold to inspect -- bees are clustered!"
  5. Dry day inspection -- On Sunny or Overcast day, inspection works normally without restrictions.
  6. Visual tints -- Observe CanvasModulate color changes. Rainy has blue tint, sunny has warm tint.
  7. Save persistence -- Note current weather. Sleep (saves). Quit and reload. Weather state persists.
  8. Temperature range -- Check dev panel for temperature. Verify seasonal ranges: Winter (12-32F), Spring (40-65F), Summer (70-86F), Fall (50-70F).
  9. Forage multipliers -- Verify forage affected by weather: Sunny 1.0x, Rainy 0.0x, Windy 0.4x, Cold 0.0x, Drought 0.3x.
Expected Results:

Weather rolls daily using season-appropriate probabilities. Temperatures accurate per NOAA Des Moines data. Rainy/Cold days prevent hive inspection with correct messages. Visual tints change smoothly. Weather persists after save/load. Forage multipliers correctly reduce/eliminate nectar collection.

Edge Cases:

Weather affects sting multiplier during inspections. Rare weather (HeatWave, Drought, Foggy) appear less frequently. Cold shows frost/snow effects. Rainy shows rain particles or wetness effect.

Prerequisites:

Starting inventory/chest contains: Hive Stands, Deep Bodies, Frames, Lids, Package Bees.

Test Steps:

  1. Open chest -- Walk to storage chest, press E. Inventory overlay shows available items.
  2. Take hive stand -- Move Hive Stand to hotbar. Close chest.
  3. Place stand -- Select Hive Stand, face open ground, press E. Stand placed on grid, snaps to position.
  4. Grid overlay -- When stand selected, blue/amber grid overlay appears showing valid placement locations.
  5. Add deep body -- Select Deep Body, press E on stand. Body snaps on top, hive visually grows taller.
  6. Add frames -- Select Frames, press E repeatedly on body. Frames added one at a time. Progress notification: "2/10 frames loaded", etc.
  7. Complete frames -- After 10 frames, notification shows "10/10 frames loaded". Hive ready for next component.
  8. Add lid -- Select Lid, press E on hive. Lid closes hive visually, structure complete (no longer open-topped).
  9. Install bees -- Select Package Bees, press E on hive. Bee swarm cloud visual appears above hive.
  10. New colony lockout -- Try to inspect hive immediately (select Hive Tool, press E). Toast: "Establishing... 7d" (7-day lockout).
  11. Advance past lockout -- Use dev mode to advance 7 days. Try inspect again. Opens normally without lockout.
Expected Results:

Each component snaps to hive in correct order without manual positioning. Hive visually grows with each component. Grid overlay shows valid placement. Frames count 1-10 accurately. Lid closes hive. Package Bees creates swarm animation. New colonies have 7-day lockout preventing inspection.

Edge Cases:

Without deep body, frames show error: "Place deep body first!" Without 10 frames, error: "10 frames loaded -- select Lid". Cannot place stand on occupied tiles. Each component consumed from inventory on placement.

Hive Inspection

Frame-by-frame colony examination with detailed statistics and queen-finding minigame. Core gameplay loop.

Prerequisites:

Hive Tool in hotbar. Hive with colony past 7-day lockout. NOT Rainy or Cold weather.

Test Steps:

  1. Select Hive Tool -- Click hotbar slot with tool or press corresponding number.
  2. Open inspection -- Press E near hive. InspectionOverlay opens showing honeycomb grid and colony details. Energy cost -10 deducted.
  3. Navigate frames -- Press A repeatedly, navigate left. Press D, navigate right. All 10 deep body frames accessible.
  4. Flip frame side -- Press F. Frame flips to reverse side (A becomes B). Press F again, flips back.
  5. Honeycomb grid -- Grid displays cells: Deep frames (70x50), Super frames (70x35). Cells color coded: brood (gold), honey (amber), pollen (orange), empty (white).
  6. Stats panel building -- As frames viewed, stats panel builds showing: population count, brood pattern, honey stores (lbs), comb condition, mite load.
  7. Knowledge tier affects accuracy -- Level 1 shows estimates, Level 5 shows exact numbers. (Dev mode forces Level 5 display.)
  8. Mark frame harvest -- On frame with capped honey, press H. Gold harvest overlay appears, marking for later extraction.
  9. Mark entire super -- Press Shift+H. All super frames marked for harvest (all show gold overlay).
  10. Queen Finder minigame -- Look for queen bee on honeycomb. Click her. 80% of time success (+15 XP). 20% of time not visible. On success, she highlights and award displays.
  11. Close inspection -- Press ESC. Overlay closes, return to gameplay. Harvest marks persist, visible on hive exterior.
Expected Results:

Inspection overlay displays accurate colony state based on hive simulation. Stats accuracy depends on knowledge tier progression. Frame navigation smooth with A/D controls. Flip changes frame side instantly. Harvest marking applies gold overlay and persists. Queen Finder provides skill-based minigame with occasional queen visibility.

Edge Cases:

Sting check on close: 25% base chance x weather mult x 0.3 (suit) x 0.2 (smoker). Energy penalty: -10 no suit, -5 with suit. Toast: "Ouch! You got stung! (-X energy)". Fermentation warning if marking <60% capped. Queen difficulty scales with level (Easy L1, Hard L5).

Smoker & Bee Suit

Protective equipment reducing sting risk during inspections. Equipment-based risk mitigation.

Prerequisites:

Access Feed & Supply in Cedar Bend ($70 minimum). Purchase equipment there.

Test Steps:

  1. Buy smoker -- Visit Feed & Supply. Navigate to Smoker ($25). Press E to purchase. Money deducted, Smoker added to inventory.
  2. Buy bee suit -- In shop, navigate to Bee Suit ($45). Press E. Money deducted, Bee Suit added to inventory.
  3. Equip bee suit -- Suit is passive. Simply having it in inventory provides protection (no separate equip needed).
  4. Select smoker -- Add Smoker to hotbar, select it.
  5. Use smoker -- Stand near hive, press E with smoker selected. Visible puff of smoke appears, hive shows smoke cloud, "_smoker_active" flag set.
  6. SMOKED indicator -- "SMOKED" orange label appears on HUD indicating current hive is smoked.
  7. Inspect smoked hive -- Open inspection immediately on that hive. Sting chance dramatically reduced compared to unsmoked.
  8. Smoker resets on close -- Close inspection. Smoker effect resets (SMOKED indicator disappears). Smoker must be re-used before next inspection.
  9. Test without equipment -- Remove both smoker and suit. Inspect hive. Note higher sting frequency, -10 energy penalty.
  10. Test suit only -- Remove smoker, keep suit. Inspect. Sting chance reduced but not as much. Energy penalty -5.
  11. Test smoker only -- Remove suit, have smoker. Use smoker, inspect. Sting chance significantly reduced. Energy penalty -10.
  12. Test both -- Have both equipped. Use smoker, inspect. Sting chance minimized. Energy penalty -5.
Expected Results:

Smoker gives visual puff effect on use. Bee suit provides passive protection when in inventory. Sting reduction: smoker=80% (x0.2), suit=70% (x0.3). Both stack multiplicatively (combined x0.06, ~94% reduction). SMOKED indicator persists until inspection closes.

Edge Cases:

Smoker resets after each inspection close, must be re-applied. Suit never wears out. Single smoker application lasts entire inspection. Weather affects sting rates independently (Cold/Rainy prevent inspection).

Harvest Yard (Full Pipeline)

5-station outdoor honey processing system converting raw honeycomb to bottled honey jars. Complete extraction workflow.

Prerequisites:

Hive with capped honey super. Frames marked for harvest during inspection. Harvest Yard on home property.

Test Steps:

  1. Locate harvest yard -- Walk east on home property (approximately 480, 350). Five stations visible: Super Pallet, Scraping Station, Honey Extractor, Bottling Table, Beeswax Collection.
  2. Remove full super -- Open Hive Management on hive. Remove super with honey (becomes ITEM_FULL_SUPER in inventory).
  3. Deposit super -- Walk to Super Pallet station, press E. Super deposited and removed from inventory.
  4. Scraping minigame -- Walk to Scraping Station, press E. Scraping minigame opens showing honeycomb grid on Side A: Click-drag mouse across cells (3-cell brush width). Scrape wax cappings until ~95% uncapped. Frame auto-flips to Side B. Repeat scraping. Both sides complete, frame moves to scraped pallet. Dev mode: brush size doubles.
  5. Extractor minigame -- Walk to Honey Extractor, press E. Speed gauge shown: Tap E rapidly to keep above 40% threshold (red line). Speed decays 0.65/sec, each E tap adds 0.22. Progress fills at 0.10/sec above threshold. Green gauge=progress, red=too slow. ~10 seconds sustained effort to complete. Beeswax accumulated (~0.735 lbs/frame) added to inventory.
  6. Bottling minigame -- Walk to Bottling Table, press E. Bottling minigame opens showing jar: Press E to toggle honey gate. When open, honey fills jar at 0.6 lbs/sec. Auto-stops when jar full (1 lb). Completed jar placed, new jar appears. Can bottle up to 40 jars/session. Press ESC to finish early; unused jars returned.
  7. Verify output -- After bottling completes, verify honey_jar items in inventory with correct count. Each jar worth $10 base.
  8. Full pipeline run -- Complete entire pipeline: deposit super -> scrape -> extract -> bottle -> collect jars. Verify no items lost between stations.
Expected Results:

Beeswax accumulated ~0.735 lbs/frame. Full pipeline seamless: deposit -> scrape -> extract -> bottle -> jars in inventory. Each minigame has clear success/failure states. No items lost. Extracted honey shows visually in jars.

Edge Cases:

ESC cancels minigame mid-progress, returns partial progress to inventory. Winter reserve warning in Fall if colony honey <60 lbs before removal. Scraping: 95% success normally, drops to 80% at >90% completion. Side auto-flips without user action.

Hive Simulation (Background)

Colony biology simulation driving hive gameplay. Brood patterns, population dynamics, honey storage, and disease.

Prerequisites:

At least one hive with installed colony past 7-day lockout. Play for multiple in-game weeks.

Test Steps:

  1. Initial state -- Inspect fresh colony (past lockout). Note population, brood pattern (initial small pattern center frames), honey stores.
  2. Advance days -- Use dev mode to advance 7 days. Reinspect same hive. Verify: Population increases. Brood pattern expands outward from center. Honey stores increase (if nectar flow).
  3. Brood pattern shape -- Verify brood follows 3D ellipsoid pattern: Queen lays center frames first. Pattern expands dome-shaped (center-out). Comb filling shows foundation -> drawn -> honey progression.
  4. Seasonal population -- Advance through seasons. Verify: Spring population grows ~12k to ~30k. Summer peaks ~40k-50k. Fall declines as brood production slows. Winter minimal ~8k-12k.
  5. Honey flow seasons -- Verify honey stores: Increase Spring/Summer (nectar flow). Plateau Fall (reduced). Slowly consumed Winter (cluster living).
  6. Super fill visuals -- Observe hive overworld sprites. Super color progression: Grey (empty), Pale yellow (partial), Gold (moderate), Amber (mostly full), Deep amber (full/capped).
  7. Mite load progression -- Advance several weeks into Fall. Inspect. Varroa mite load increases over time (visible speckling in brood). Check dev panel for exact count.
  8. Forage status -- Verify forage status changes by season/weather: Abundant (Spring/Summer good), Adequate (Fall), Stressed (Winter/cold), Dearth (very poor).
  9. Dev mode full stats -- Press G, open dev panel. Verify immediate full hive statistics (exact values, not estimates). Knowledge tier irrelevant in dev mode.
Expected Results:

Colony population follows realistic seasonal curve. Brood pattern expands dome-shaped from center. Honey stores increase during nectar flow, consumed during dearth. Mite load increases progressively. Super fill color accurately represents contents. Forage status reflects season/weather.

Edge Cases:

Season factor (0.05-1.0) drives foraging rates. Package bees: 7-day lockout, 4-6 day laying delay. Maximum population limited by space. Heavy mite load visibly damages brood pattern. Dev mode exposes all hidden simulation values.

ESC / Pause Menu

In-game pause overlay with settings and music controls. Main gameplay pause interface.

Prerequisites:

In gameplay (not main menu). Any location (home property, Cedar Bend, County Road).

Test Steps:

  1. Open pause menu -- Press ESC or P key. Pause menu appears with slide-in animation.
  2. Game pauses -- Verify game state frozen: No movement, no time progression, no animations (except menu).
  3. Status display -- Pause menu shows: Season name, Day count, Year count, Money amount, Energy percentage, Level/standing tier.
  4. Main buttons -- Four buttons appear: "Resume" (unpause), "Save" (save game), "Settings" (audio/graphics), "Exit" (quit with confirmation).
  5. Click Resume -- Clicks "Resume". Menu closes, game unpauses, time/animations resume.
  6. Click Save -- Reopen menu, click "Save". "Game saved!" confirmation appears. Game remains paused.
  7. Music volume slider -- In pause menu, locate "Music Volume" slider. Adjustable. Verify music volume changes in real-time.
  8. SFX volume slider -- Locate "SFX Volume" slider. Adjust it.
  9. Master volume slider -- Locate "Master Volume" slider. Adjust to 50%. Music volume reduced to half.
  10. Mute Music -- Check "Mute Music". Music stops. Uncheck, music resumes.
  11. Mute auto-uncheck on slider -- Check Mute Music. Drag Music Volume slider >0%. Mute auto-unchecks, music resumes.
  12. Open Settings -- Click "Settings". Audio Settings panel opens showing: Music/SFX/Master sliders, Mute checkboxes.
  13. Adjust settings -- In settings panel, adjust sliders. Verify sync with main panel (bidirectional).
  14. Return from settings -- Press ESC from settings. Return to main pause menu (no double-close).
  15. Click Exit -- Click "Exit". Confirmation dialog appears: "Save before exiting?" with "Cancel", "Save & Exit", "Exit Without Saving".
  16. Cancel exit -- Click "Cancel". Return to pause menu (no quit).
  17. Save & Exit -- Click "Save & Exit". Game saves, quits to main menu. Verify save file written.
  18. Exit Without Saving -- On next play, open pause -> Exit -> "Exit Without Saving". Quit immediately without saving.
  19. Rapid ESC presses -- Press ESC multiple times rapidly. Menu appears once, doesn't break or duplicate.
  20. ESC closes menu -- From open pause menu, press ESC. Menu closes, game unpauses (same as Resume).
  21. Menu z-index -- Verify pause menu appears above gameplay but below error dialogs. "[ESC] to close" hint visible at bottom.
Expected Results:

Pause menu appears with slide-in animation. Game state completely frozen. Status shows accurate character/game data. All buttons functional. Resume unpauses, Save triggers write, Settings opens sub-panel, Exit shows confirmation. Settings sliders bidirectionally synced. Music controls responsive. Confirmation prevents accidental quit.

Edge Cases:

Rapid ESC presses don't break state. Menu at layer 30 (above most overlays). May not appear during cutscenes/certain minigames. Save button may be disabled if recently saved (prevent spam). Volume conversion logarithmic dB (-80dB to 0dB).
