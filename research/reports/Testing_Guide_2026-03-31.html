1Main Menu & Start Modes

What it is: The newly rewritten title screen and game mode selection. The menu features a centered logo, four main action buttons, music playback, and fade-in animation. Players choose between starting in Spring (Day 1) or Fall (Day 113, with 2 filled supers), continuing an existing save, or quitting.

Prerequisites: Launch the game from Godot or executable (project.godot MainMenu scene). Start on a clean save (no existing save file) to test disabled Continue button.

Test Steps:

  1. Launch game — verify MainMenu scene (res://scenes/ui/MainMenu.tscn) loads
  2. Verify dark warm background with subtle center glow displays (no hex dot pattern)
  3. Verify game logo (game_logo.png) appears prominently centered in upper portion (160x80 viewport pixels)
  4. Verify logo loads dynamically via Image.load_from_file() (not from Godot import cache)
  5. Verify four buttons are vertically centered below logo:
    • Start in Spring (green accent) — Day 1, Quickening, standard new game
    • Start in Fall | 2 Full Supers (orange accent) — Day 113, Full-Earth, A-grade colony
    • Continue (amber) — should be greyed/disabled if no save file exists
    • Quit (muted color)
  6. Verify thin amber decorative dividers above and below button group
  7. Verify version line at bottom: "v0.1 dev -- Cedar Bend, Iowa"
  8. Verify fade-in animation plays on load (0.8 seconds)
  9. Verify main theme music plays via MusicManager.play_title_theme() (mainTheme.mp3)
  10. Press ESC key — verify game quits cleanly from main menu
  11. Click "Start in Spring" — character creator should load, verify Day 1, Quickening season
  12. Sleep (Z key) to return to menu, click "Start in Fall | 2 Full Supers"
  13. Confirm character — verify spawn on home property, Day 113, Full-Earth season, A-grade colony with 2 filled/capped supers
  14. Create another save, then sleep to return to menu
  15. Verify "Continue" button is now active (not greyed) and loads the most recent save (correct day/inventory/money)
Expected Results: Logo displays correctly, all four buttons visible and properly spaced, music plays without stutter. Spring/Fall start modes create correct initial states. Continue button is greyed when no save exists, active when save exists. ESC key quits cleanly. Fade-in animation smooth. Version text displays correctly.
Known Edge Cases / Things to Watch:
  • Verify logo file path is correct and image loads without file-not-found errors
  • Check image loading via Image.load_from_file() works and doesn't crash on missing file
  • Verify Continue button enable/disable state matches save file existence
  • Test rapid button clicks — ensure no duplicate scene loads
  • Verify music transitions smoothly (no stutter at fade-in end)

Prerequisites: Complete main menu (New Game selected, not Continue).

Test Steps:

  1. Name input: Type a name using keyboard (e.g., "Alex", "Jordan") — verify text appears and is stored
  2. Pronouns: Press A / D to cycle through options (She/Her, He/Him, They/Them) — verify selection changes

  5. Press E to confirm and start game

  • Verify backstory text doesn't truncate on screen
  • Confirm E key doesn't get stuck if pressed multiple times rapidly

3Player Movement & Controls

What it is: Core player locomotion and input handling. Includes 4-directional movement, running, interactions, hotbar switching, and mode toggles.

Prerequisites: In-game on home property (any season/day). Player should be able to move freely.

Test Steps:

  1. WASD movement: Press each key individually — verify player moves in correct direction (W=up, A=left, S=down, D=right) at normal speed
  2. 8-way facing: Hold W+D (diagonal) — verify player faces NE direction and moves diagonally
  3. Running: Hold Shift + W — verify player moves faster than normal
  4. Interact (E key): Stand near a chest or hive and press E — verify interaction occurs (chest opens, etc.)
  5. Hotbar scroll: Hold an item in hotbar, use mouse wheel up/down — verify selection cycles through slots
  6. Hotbar direct select: Press 1, 2, 3, etc. — verify hotbar slot changes
  7. Till mode: Press T — verify label appears above player (e.g., "[Till Mode]"), press again to toggle off
  8. Plant mode: Press F — verify label appears, press again to toggle off
  9. Rotate: With a placeable item selected, press R — verify rotation indicator or behavior (if applicable)
  10. Sleep: Press Z — verify day advances, game saves, and player wakes next morning
  11. Info panel: Press Tab — verify detailed info overlay appears, press again to hide
  12. Map: Press M — verify map overlay opens with POI markers
  13. Dev mode: Press G — verify dev panel appears at bottom (if enabled), press again to hide
  14. Pause menu: Press Esc or P — verify pause overlay opens, press again to resume
Expected Results: All keys respond immediately and accurately. Movement is smooth and responsive. Mode toggles (till, plant, dev) clearly indicate active state. No input lag or double-firing.
Known Edge Cases / Things to Watch:
  • Test key mashing (rapid presses) — ensure no input queue/buffer issues
  • Test moving off-screen edge — verify scene doesn't scroll or crash
  • Test interaction while moving (press E while holding WASD) — should prioritize interaction
  • Verify hotbar number keys (1-9) don't conflict with other UI

4HUD (Heads-Up Display)

What it is: The persistent on-screen information display, including time, resources, stats, and status indicators.

Prerequisites: In-game on home property. Be awake (not in a menu).

Test Steps:

  1. Top bar: Verify visible at top of screen with season icon, month name (e.g., "March"), day number, and time of day
  2. Weather label: Check displays current weather (e.g., "Clear", "Rainy", "Cold"). Change weather via Dev Mode (G) and verify label updates
  3. Money display: Note starting money amount, make a purchase or sale, verify money updates in HUD
  4. Energy bar: Perform actions (chop tree, inspect hive) — verify bar drains with each action, then refills on sleep (Z)
  5. Level + XP bar: Perform XP-granting actions (complete quest, harvest) — verify bar fills gradually, level increases on full bar
  6. Community Standing: Verify label shows current standing (starts at "Stranger"), interact with NPCs to change standing and confirm label updates
  7. Hotbar: Select various items using number keys (1-9) — verify hotbar highlights correct slot and shows item count
  8. Active item label: Verify currently selected item name displays (e.g., "[Hive Stand]")
  9. SMOKED indicator: Use smoker on hive, verify orange "[SMOKED]" label appears in HUD (disappears after inspection)
  10. Tab info panel: Press Tab — verify detailed stats panel expands, showing full resource/time/standing info, press again to collapse
Expected Results: All HUD elements present and updating correctly in real-time. Indicators change immediately upon action. Tab overlay provides detailed info without lag.
Known Edge Cases / Things to Watch:
  • Test with very long player names — verify HUD text doesn't overflow or clip
  • Test extreme values (e.g., $999999 money, Level 99) — ensure no display glitches
  • Verify Tab overlay doesn't block critical game elements
  • Check SMOKED indicator disappears immediately after inspection closes

6Weather System

What it is: Dynamic weather states (Clear, Cloudy, Rainy, Cold) that affect gameplay, particularly hive inspection.

Prerequisites: In-game with a hive that is ready for inspection (colony installed, past 7-day lockout). Dev Mode enabled (G) to control weather.

Test Steps:

  1. Observe current weather in HUD and Dev panel
  2. Use Dev Mode to set weather to "Rainy"
  3. Attempt to inspect hive (select hive tool or E near hive) — verify toast appears: "Too wet to open hives -- wait for dry weather!"
  4. Verify inspection does NOT proceed, hive remains closed
  5. Change weather to "Cold" via Dev Mode
  6. Attempt to inspect — verify toast appears: "Too cold to inspect -- bees are clustered!"
  7. Verify inspection blocked, hive closed
  8. Change weather to "Clear" or "Cloudy"
  9. Attempt to inspect — verify inspection proceeds normally, no toast warning
  10. Sleep to save, quit to menu, continue game — verify weather state persists
Expected Results: Rainy/Cold weather blocks inspection with appropriate toast messages. Clear/Cloudy weather allows normal inspection. Weather persists through save/load.
Known Edge Cases / Things to Watch:
  • Test rapid weather changes (Dev Mode) — ensure toast doesn't stack
  • Verify toast message disappears after 2-3 seconds (not permanent)
  • Check that other activities (harvesting, moving, trading) are not blocked by Rainy/Cold

7Hive Building (Step-by-Step)

What it is: The complete process of constructing a hive from raw components (stand, body, frames, lid) and installing a package of bees.

Prerequisites: Start in Spring mode. Home property visible. Storage chest contains: Hive Stand, Deep Body, Frames (10 required), Lid, Package Bees.

Test Steps:

  1. Open chest: Stand near storage chest and press E — verify chest opens, shows inventory
  2. Take Hive Stand: Select Hive Stand from chest, add to hotbar
  3. Place stand: Stand on open ground, select Hive Stand in hotbar, press E — verify Hive Stand placed, grid overlay shows valid placement
  4. Take Deep Body: Return to chest, take Deep Body to hotbar
  5. Place body: Select Deep Body, stand on/near stand, press E — verify Deep Body stacks on stand
  6. Take Frames: Return to chest, take Frames to hotbar
  7. Load frames: Stand at hive, select Frames, press E repeatedly — each press adds one frame (count in HUD/info panel). Continue until 10 frames loaded
  8. Take Lid: Return to chest, take Lid to hotbar
  9. Place lid: Select Lid, stand at hive, press E — verify hive structure now complete (model updates, hive now shows "Unlocked" state)
  10. Take Package Bees: Return to chest, take Package Bees to hotbar
  11. Install bees: Select Package Bees, stand at hive, press E — verify package installed, hive switches to "Establishing..." state, 7-day lockout begins
  12. Verify lockout: Attempt to inspect hive (press E near hive) — verify message shows "Establishing... Xd" (X = days remaining)
  13. Sleep (Z) 7 times — each day, remaining count decrements. After 7 days, verify count is 0 and hive can be inspected normally
Expected Results: Each building step succeeds in correct sequence. Grid overlay validates placement. Hive structure updates visually. Package install triggers 7-day lockout with countdown. On day 8, lockout ends and inspection is possible.
Known Edge Cases / Things to Watch:
  • Test placing stand on invalid ground (water, tree) — should show "invalid placement" and not place
  • Test loading more than 10 frames — should reject or cap at 10
  • Test building without all components (e.g., missing frames) — lid should not place on incomplete hive
  • Test moving/rotating hive after partial build — should either lock parts together or prevent movement

9Hive Inspection

What it is: The primary gameplay loop. Opens an overlay to view individual frames, check bee health, and mark frames for harvest. Includes sting mechanics and bee suit/smoker reductions.

Prerequisites: Hive with colony installed and past 7-day lockout. Clear or Cloudy weather (not Rainy or Cold). Player has energy > 0. Optional: Bee suit in inventory, smoker in hotbar.

Test Steps:

  1. Select hive tool: Equip hive tool (or no item) in hotbar
  2. Open inspection: Stand near hive, press E — verify InspectionOverlay opens (costs energy, verify energy bar decreases by appropriate amount)
  3. Navigate frames: Press left/right arrow keys — verify you move between frames (frame 1, 2, 3, etc.)
  4. View super: If hive has multiple supers, press up/down arrows — verify you navigate between supers
  5. Frame stats: Verify stats panel on right updates as frames are viewed (shows bee count, pollen, honey, capped brood, etc.). Stats should accumulate/progress as you move
  6. Mark frame: While viewing a frame, press H — verify frame is marked for harvest (indicator appears: checkmark, highlight, or label)
  7. Unmark frame: Press H again on marked frame — verify it's unmarked
  8. Mark super: Press Shift + H — verify entire super is marked (all frames in super show harvest indicator)
  9. Sting roll (no protection): Close inspection and open it again. Repeat 5 times. Count how many times you get stung (toast: "Ouch! You got stung! (-X energy)"). Average should be ~25% (1-2 stings per 5 inspections)
  10. Equip bee suit: Add bee suit to inventory (from chest or purchase from shop)
  11. Sting roll (with suit): Open inspection 5 times while wearing bee suit. Verify stings are much rarer (expected ~10% instead of 25%)
  12. Smoker protection: Take smoker to hotbar, stand near hive, press E with smoker selected — verify puff animation/toast, "[SMOKED]" label appears. Open inspection immediately — verify stings very rare (expected <5%)
  13. Daily quest: Open inspection for the first time that in-game day — verify "daily_inspection" quest completes (notification/reward)
  14. Close inspection: Press ESC — verify overlay closes, player can move again
Expected Results: Inspection opens and closes smoothly. Frame navigation works. Stats display correctly. Marking/unmarking works reliably. Sting chances match expected percentages (25% base, x0.3 with suit, x0.2 with smoke). Daily quest fires once per day. No crashes or stat display errors.
Known Edge Cases / Things to Watch:
  • Test opening inspection with 0 energy — should either block or warn before opening
  • Test sting chance with BOTH suit and smoker — should stack multiplicatively (25% * 0.3 * 0.2 = 1.5%)
  • Test marking frames, then leaving hive and returning — marked frames should persist
  • Test rapid ESC presses — panel should close cleanly on first press
  • Test with hive that has 3+ supers — verify navigation is smooth, no index errors
  • Test smoker effect — it should reset after inspection (SMOKED indicator disappears), requiring re-puff if inspecting again

10Smoker & Bee Suit

What it is: Equipment items that reduce sting chance during inspection. Smoker must be used before each inspection; bee suit is passive/persistent.

Prerequisites: Have or purchase from Feed & Supply: Smoker ($25), Bee Suit ($45). Have a hive ready for inspection.

Test Steps:

  1. Obtain smoker: Buy from Feed & Supply or take from chest — verify it appears in inventory
  2. Equip smoker: Add smoker to hotbar (number key 1-9)
  3. Use smoker: Stand near hive, select smoker, press E — verify puff animation/sound, toast appears (e.g., "Bees calmed!"), "[SMOKED]" label appears in HUD
  4. Check HUD: Verify orange "[SMOKED]" indicator visible in HUD
  5. Inspect with smoke: Open hive inspection immediately (within same frame) — verify sting chance greatly reduced
  6. Close inspection: After inspection closes, verify "[SMOKED]" indicator disappears from HUD
  7. Re-inspect without re-smoking: Open inspection again — verify smoker effect is gone (stings return to normal 25% chance)
  8. Obtain bee suit: Buy from Feed & Supply or take from chest
  9. Equip bee suit: Bee suit should be automatically equipped when in inventory (or select from menu) — verify it appears as worn item in inventory/status
  10. Inspect with suit: Open hive inspection while wearing suit, no smoker — verify sting chance reduced (expected ~10%)
  11. Inspect with suit + smoke: Use smoker, then inspect while wearing suit — verify sting chance very low (expected ~1-2%)
  12. Remove suit: Drop suit from inventory or unequip — verify sting chance returns to ~25% on next inspection
Expected Results: Smoker provides temporary protection (one inspection per use). Bee suit provides persistent protection while equipped. Both stack multiplicatively. HUD clearly indicates SMOKED status, which clears after inspection. Equipment purchases deduct money correctly.
Known Edge Cases / Things to Watch:
  • Test using smoker multiple times in a row — verify no duplicate SMOKED status
  • Test opening inspection then using smoker (should be too late) — verify protection doesn't apply retroactively
  • Test bee suit in other activities (tree chopping, harvesting) — should not affect those (sting mechanics only apply to inspection)
  • Test dropping bee suit during inspection — suit protection should persist for that inspection, not carry to next

11Harvest Yard (Full Pipeline)

What it is: End-game honey production facility with 5 stations: super pallet, scraping, extractor, and bottling. Converts marked frames into bottled honey and beeswax.

Prerequisites: Hive with at least one super that has capped honey. Frames in that super marked for harvest (H key during inspection). Location: East of home property (~x=480, y=350). Clear weather or safe to work outside.

Test Steps:

  1. Travel to Harvest Yard: Exit home property east — verify you enter Harvest Yard area and see 5 station structures
  2. Super pallet station: Select full super from hotbar (take from hive management), walk to Super Pallet, press E — verify super deposits on pallet, inventory updates
  3. Scraping station: Approach scraping station, press E — verify scraping minigame opens (24x14 hex grid)
  4. Scrape Side A: Click+drag mouse over cells (3-cell brush) to uncap them. Uncap ~70% of cells — verify progress bar fills
  5. Side A complete: When enough cells uncapped, verify frame flips automatically to Side B
  6. Scrape Side B: Repeat uncapping on back side. When complete, verify frame moves to "Scraped Frame Pallet"
  7. Scraped pallet: Approach scraped pallet, press E — verify scraped frames can be collected (added to inventory)
  8. Extractor station: With scraped frame, press E at extractor — verify extractor minigame opens (vertical speed gauge)
  9. Extractor effort: Press E repeatedly to keep speed gauge above 60% (red threshold). Sustain for ~20 seconds — gauge should show green (progress), red (falling behind)
  10. Extractor complete: When timer reaches 0, verify frame processing complete. Beeswax should accumulate in inventory (~0.735 lbs/frame)
  11. Honey bucket: Approach bottling table/honey bucket, press E — verify honey gate toggles, honey starts flowing into jar
  12. Fill jar: Watch jar fill visually (should take ~10 seconds at 0.6/sec flow rate). When full, verify auto-stops filling
  13. Stack jars: Continue collecting honey — up to 40 jars should stack visibly on table
  14. Collect jars: Press E to collect — verify jars added to inventory
  15. Return to home: Travel back west — verify all collected honey and beeswax persist in inventory
  16. Winter warning (Fall): In Fall season, if removing a super with < 60 lbs stores, verify warning toast appears during step 2
Expected Results: All 5 stations function smoothly in sequence. Scraping minigame is playable and gives visual feedback. Extractor requires sustained effort. Bottling visual effect is smooth. Final inventory shows honey jars + beeswax. No crashes or soft-locks.
Known Edge Cases / Things to Watch:
  • Test scraping with very fast clicking — ensure grid doesn't glitch or allow uncapping outside grid
  • Test extractor with minimum effort — gauge should drop into red zone, progress slow
  • Test leaving extractor mid-minigame (ESC) — should either cancel or auto-complete partial work
  • Test collecting from honey bucket when empty — should show "empty" message or not allow collection
  • Test with unmarked frames (didn't mark with H) — should either show warning or not appear in harvest yard

14Tree Chopping Minigame

What it is: Mini-game to fell trees and gather logs. Requires axe and uses space bar + timing mechanics.

Prerequisites: Purchase axe from Feed & Supply ($15) or obtain from chest. Travel to County Road zone (from Cedar Bend, continue east or check map). Have at least 15 energy.

Test Steps:

  1. Navigate to tree: Walk around County Road and locate a standing tree
  2. Interact without axe: Without axe selected, walk to tree, verify hint shows "[E] Chop (Need Axe!)" or similar
  3. Equip axe: Select axe in hotbar
  4. Interact with axe: Stand near tree, press E — verify Chopping minigame overlay opens
  5. Minigame UI: Verify power meter is visible (oscillating bar), shows green sweet spot, yellow OK zone, red miss zone
  6. Swing axe: Press Space to swing — verify power meter position registers (hit sweet spot = good, yellow = OK, red = miss)
  7. 7 swings required: Continue swinging. On successful hits, tree shows damage/cracks. Meter speed increases with each swing. Required hits: 7 total
  8. Sweet spot moves: Verify sweet spot moves position slightly after each swing (increasing difficulty)
  9. Tree fells: After 7 hits, verify tree falls, stump remains, logs appear in inventory (3-5 logs based on accuracy)
  10. Energy cost: Verify energy decreased by 15
  11. XP reward: Verify +15 XP awarded (check XP bar)
  12. Tree respawn: Sleep for 21 in-game days — verify stump regrows into full tree, ready to chop again
Expected Results: Minigame opens cleanly. Meter mechanics are responsive and fair. 7 hits fell tree. Logs awarded based on accuracy (min 3, max 5). Energy/XP deducted/awarded correctly. Trees respawn after 21 days. No crashes or soft-locks.
Known Edge Cases / Things to Watch:
  • Test failing hits (red zone) — tree should require more total swings to fell (not instant fail, but penalty)
  • Test quitting minigame mid-swing (ESC) — should cancel, refund energy, no logs awarded
  • Test chopping with low energy (< 15 remaining) — should either block or warn before opening
  • Test rapid spacebar mashing — meter should not skip/glitch, only respond to actual presses
  • Test with multiple trees on screen — chopping one should not affect others

15Workbench Crafting

What it is: Crafting station where player converts logs to lumber and crafts hive components (frames, super boxes, deep bodies, etc.).

Prerequisites: Have hammer in inventory (buy from Feed & Supply for $10 or take from chest). Have logs from tree chopping. Location: Home property workbench (~x=520, y=-20 area).

Test Steps:

  1. Navigate to workbench: From home property, locate workbench structure (north side, near tree line)
  2. Open workbench: Stand near workbench, press E — verify Workbench UI opens
  3. Resource display: Verify resource section shows: logs available, lumber available, "[Hammer OK]" or "[No Hammer!]" status
  4. Convert logs: With logs in inventory and hammer equipped, press C — verify "3 logs -> 1 lumber" conversion happens, resource display updates
  5. Browse recipes: Press up/down arrows to browse available recipes (Frames, Super Box, Deep Body, Hive Stand, Lid)
  6. Recipe details: Each recipe shows: name, required lumber, output (e.g., "Frames (10-pack): 3 lumber required")
  7. Craft frames: Select Frames (requires 3 lumber), verify you have lumber, press Enter — verify 10-pack of frames added to inventory, lumber deducted
  8. Craft super box: Select Super Box (requires 4 lumber), press Enter — verify super box added, lumber deducted
  9. Craft without hammer: Drop hammer from inventory, try crafting — verify craft fails with message "Cannot craft without hammer!"
  10. No lumber: Spend all lumber, try crafting — verify appropriate message appears (e.g., "Not enough lumber")
  11. Close workbench: Press ESC — verify UI closes, player can move freely
Expected Results: Workbench UI opens/closes smoothly. Log-to-lumber conversion works (3:1 ratio). All recipes available and functional. Crafting deducts resources correctly. Hammer requirement enforced. No crashes or item duplication.
Known Edge Cases / Things to Watch:

16Quest System

What it is: Task-based rewards system. Daily quests reset each day; story quests are one-time. Each quest completion grants XP and money.

Prerequisites: In-game on home property. Access NPCs (Uncle Bob on property), hive ready for inspection, honey for trading.

Test Steps:

  1. bob_intro quest: Locate Uncle Bob NPC on home property. Interact with him (press E) — verify quest triggers and immediately completes, grants +25 XP
  2. daily_inspection quest: Sleep to ensure it's a new day (Day counter increments). Open hive inspection for the first time that day — verify quest completes, grants +10 XP, +$5
  3. Re-inspect same day: Open same hive again (no new day) — verify daily_inspection quest does NOT trigger again (already completed today)
  4. Sleep (new day): Press Z to advance to next day — verify daily_inspection quest resets (available for completion again)
  5. daily_harvest quest: Complete one full super harvest at Harvest Yard (deposit super, scrape, extract, bottle) — verify quest completes on bottling final jar, grants +15 XP, +$10
  6. daily_market quest: On Saturday (non-Winter), sell honey to Frank at Cedar Bend market — verify quest completes on sale, grants +20 XP
  7. Verify XP bar: After each quest, verify XP bar in HUD increases by appropriate amount. When bar fills, verify level increases and bar resets
  8. Quest tracking: Check if quest status is visible somewhere (HUD, info panel, or quest log) — track completion status
Expected Results: All quests trigger at correct moments. Daily quests reset on new day. One-time quests (bob_intro) don't repeat. XP/money rewards apply correctly. Quest completion visibly updates level/XP bar.
Known Edge Cases / Things to Watch:
  • Test completing quests out of expected order (e.g., market before inspection) — should still work
  • Test with very high XP (near level cap) — ensure level increases work at extreme values
  • Test completing multiple quests same day — XP should stack
  • Test quest reset edge case: complete daily_inspection at 23:59 game time, sleep at 00:01 — should still be new day and quest available

17Save & Load System

What it is: Game state persistence. Sleeping saves progress; loading restores exact game state (inventory, day, money, hive status).

Prerequisites: In-game after character creation. Play for a session (do various actions: move items, earn money, advance days, build/modify hive).

Test Steps:

  1. Baseline: Note current values in HUD/info: day, season, money, inventory count, energy level
  2. Take actions: Spend 20 minutes playing: move items around, buy/sell items, harvest, inspect hive, advance multiple days
  3. Check state: Before sleeping, take a screenshot/note of: money, day, season, inventory items, hive modifications (supers, queen excluder)
  4. Sleep to save: Press Z to sleep — verify day advances, game saves (no explicit save screen needed, but confirm progression)
  5. Quit to menu: Press Esc, select Quit or exit to main menu
  6. Load via Continue: On main menu, click "Continue" button — verify game loads
  7. Verify all state: Check that all noted values match exactly:
    • Day, season, time match
    • Money matches
    • Inventory items match (same items, same quantities)
    • Hive structure matches (same supers, queen excluder, colony status)
    • Character name/pronouns from PlayerData match
  8. Multiple saves: Play session 1, save, quit. Start new game (Spring mode), play session 2, save, quit. Load session 1 Continue — verify correct save loads (not session 2 data)
Expected Results: All game state persists perfectly through save/load cycle. No data loss or corruption. Multiple saves don't interfere. PlayerData (character info) persists. Hive state persists without drift.
Known Edge Cases / Things to Watch:
  • Test inventory overflow — ensure extra items don't get lost on save
  • Test very high money amount (e.g., $999999) — ensure display doesn't overflow, value persists
  • Test loading while in a minigame/overlay (ESC then quit) — ensure state saves cleanly
  • Test multiple quit/load cycles (5+) — data should remain stable
  • Test loading save made in different zone (Cedar Bend) — ensure correct zone loads, player position correct

18Zone Transitions / Navigation

What it is: Moving between game areas (home property, Cedar Bend, County Road, Harvest Yard). Includes zone loading, NPC/object placement, and map navigation.

Prerequisites: In-game on home property. Map accessible (M key). Multiple zones available (Cedar Bend, County Road, Harvest Yard).

Test Steps:

  1. Home to Cedar Bend: Walk to east edge of home property — verify zone exit trigger, Cedar Bend loads, no black screens
  2. Player position: Verify player spawns on west side of Cedar Bend (near exit back to home)
  3. Visible elements: Verify Frank NPC present, shop building visible, market stalls visible (if Saturday/non-Winter)
  4. Cedar Bend to Home: Walk to west edge of Cedar Bend — verify transition back to home property, player spawns on east side
  5. Cedar Bend to County Road: From Cedar Bend, walk east — verify County Road loads, different terrain/trees visible
  6. County Road to Cedar Bend: Walk west from County Road — verify transition back, correct positions
  7. Map overlay (M): Press M to open map — verify shows all zones, POI markers (Harvest Yard, Workbench, Cedar Bend, etc.)
  8. No crashes: Rapidly transition between zones 10+ times — verify no crashes, slowdowns, or duplicate objects
  9. NPC persistence: Transition zones multiple times, verify NPCs appear/disappear correctly (Frank only in Cedar Bend, Uncle Bob in home, etc.)
Expected Results: Zone transitions are smooth and fast. No black screens or loading stalls. NPCs appear in correct zones. Player spawn position is logical. Map shows all areas. No memory leaks or duplicate spawns.
Known Edge Cases / Things to Watch:

21ESC / Pause Menu

What it is: In-game pause menu accessed via ESC or P key. Displays game status (season, day, year, money, energy, level), allows resuming, saving, adjusting audio, and quitting. Features music volume and mute controls directly on the main panel.

Prerequisites: In-game, actively playing (not on main menu). Game is not already paused.

Test Steps:

  1. Open with ESC: During gameplay, press Esc — verify pause menu opens and game tree pauses (get_tree().paused = true)
  2. Open with P: From gameplay, press P — verify pause menu opens (alternative to ESC)
  3. Verify "PAUSED" title bar appears at top of panel
  4. Verify status line 1 displays: Season - Day X - Year Y (e.g., "Spring - Day 5 - Year 1")
  5. Verify status line 2 displays: $money | E:energy% | level title (e.g., "$2000 | E:75% | Homesteader")
  6. Verify five buttons visible:
    • Resume — closes menu, unpauses game
    • Save — calls SaveManager.save_game()
    • Settings — opens Audio Settings sub-panel
    • Quit Game — returns to main menu
  7. Verify music volume slider visible directly on main panel (0-100%, labeled "Music Volume")
  8. Verify Mute Music checkbox visible directly on main panel
  9. Verify [ESC] to close hint text at bottom of panel
  10. Test music slider: Drag slider from 0% to 50% — verify music volume decreases in real-time
  11. Drag slider to 100% — verify music returns to full volume
  12. Test mute checkbox: Check "Mute Music" — verify music stops immediately
  13. Uncheck "Mute Music" — verify music resumes at previous volume level
  14. Test auto-unmute: With music muted, drag slider above 0% — verify mute checkbox auto-unchecks
  15. Click Settings button — verify sub-panel opens showing Music/SFX/Master sliders and Mute checkbox
  16. Test bidirectional sync: In Settings, drag Music slider to 30% — close Settings, verify main panel slider also shows 30%
  17. On main panel, check Mute checkbox — open Settings, verify Mute checkbox is also checked
  18. Verify slide-in animation when pause menu opens (smooth entry from top)
  19. Click Resume — verify pause menu closes, game unpauses and resumes normally
  20. Open pause menu again, click Save — verify SaveManager.save_game() is called and game saves
  21. Click Quit Game — verify return to main menu
  22. On main menu, press Esc — verify ESC quits to desktop (not opens pause menu)
Expected Results: Pause menu opens immediately on ESC or P. Status displays correctly. Resume button unpauses. Music slider adjusts volume in real-time. Mute checkbox stops/resumes music. Settings sub-panel syncs bidirectionally. Save button saves game. Quit returns to menu cleanly. Animation smooth. ESC on main menu quits game.
Known Edge Cases / Things to Watch:
  • Test rapid ESC presses — ensure pause menu doesn't open multiple times or flicker
  • Test opening pause menu then immediately pressing ESC again — should close immediately
  • Test dragging slider to 0% then clicking mute checkbox — verify state is consistent
  • Test Settings sub-panel with changes, then close without clicking a close button — verify changes persist
  • Test pause menu during cutscenes or dialogue — verify pause works correctly if tree is not already paused
  • Test loading a save game while pause menu is open — should close menu and load game
  • Verify status text updates if game time/resources change in pause menu (unlikely but test for edge cases)
