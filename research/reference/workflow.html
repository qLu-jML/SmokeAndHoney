Phase 01 Project Setup & Architecture
Scaffold project structure
Both ▼
Set up folders, config files, engine choice, and repo. Cowork organizes your local files; Code initializes the project.
  • Open Cowork → point it at your project folder → ask it to create the folder structure from your dev plan
  • Switch to Claude Code → run init commands (npm init, git init, etc.) and install dependencies
  • Cowork: generate a README.md and .gitignore from your GDD details
Asset pipeline planning
Cowork ▼
Organize sprite sheets, audio files, and fonts into a structured /assets directory.
  • Cowork: "Scan my assets folder and organize files by type — sprites, audio, fonts, maps"
  • Cowork: generate a manifest.json listing all assets with paths
  • Cowork: flag any duplicate or suspiciously named files
Phase 02 Core Gameplay Systems
Hive management mechanics
Claude Code ▼
Core loops: bee population, honey production, hive health, queen bee logic.
  • Claude Code: paste your GDD's mechanics section → "implement the hive state machine"
  • Claude Code: write unit tests for population growth and honey yield formulas
  • Claude Code: run tests, fix failures, iterate until green
Season & weather system
Claude Code ▼
Time progression, seasonal modifiers on bee behavior and flower availability.
  • Claude Code: implement a game clock and season enum
  • Claude Code: wire season modifiers into hive production rates
  • Claude Code: add weather events (rain, drought) with probability tables from your GDD
Balance tuning & data tables
Both ▼
Numeric values for production rates, costs, upgrades. Cowork manages the spreadsheets; Code consumes them.
  • Cowork: extract all numeric constants from your GDD into a balance.xlsx spreadsheet
  • Adjust values in the spreadsheet by hand or with Cowork's help
  • Claude Code: read balance.xlsx (or export to JSON) and wire values into game config

Upgrade tree

Dialogue & tutorial copy

Bug triage & QA pass
Both ▼
Systematic review of issues, test coverage, and final polish before release.
  • Cowork: read your bug log / notes and produce a prioritized fix list
  • Claude Code: work through the list — fix, commit, repeat
  • Cowork: update CHANGELOG.md and bump version in package.json after each session
Rules of thumb
Use Cowork when

You need to read, write, or reorganize files — docs, JSON configs, spreadsheets, markdown. Also for research, drafting copy, and any task where you want to walk away and come back to a finished artifact.

Use Claude Code when

You're writing or debugging actual source code, running builds/tests, executing terminal commands, or doing anything that requires a live runtime environment.

Handoff pattern

Cowork produces structured data (JSON, XLSX, markdown). Claude Code consumes it. Think of Cowork as your "producer" and Claude Code as your "builder."

One task at a time

Give each tool a single clear deliverable per session. "Build the hive state machine and write tests" works. "Build the whole game" doesn't.
